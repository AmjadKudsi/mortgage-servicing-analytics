Single-snapshot limitation.
The dataset is a single monthly snapshot, not a time series. True forward prediction (train on month N, predict month N+3) requires sequential monthly data. The origination model predicts current delinquency status from borrower profile — valuable for risk stratification but not a deployment-ready forecasting tool.
Class imbalance (99:1).
Only ~1% of loans are delinquent. Precision is inherently low (many false positives). This is addressed with balanced class weights and evaluated using lift metrics rather than raw precision. In production, the model would rank loans for prioritized outreach, not make binary approve/deny decisions.
What would improve it.
Monthly time-series data would enable true forecasting. Additional features (employment status, payment amounts, forbearance history) would increase AUC by an estimated 5-10 points. Ensemble methods or gradient boosting could capture more complex risk interactions. A calibration layer would make probability outputs more reliable for dollar-loss estimation.
Why the dual-model design matters.
The first attempt produced AUC = 1.0 due to data leakage — behavioral features (payment history) encoded the target variable. Rather than hiding this, the redesign separates concerns: Model B uses origination features for honest prediction (AUC ~0.77), Model A uses behavioral features for risk segmentation. This distinction — knowing when a model is predicting vs. merely classifying — is the kind of analytical judgment mortgage analytics teams value.
